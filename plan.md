## 현재 제품 상태 요약

- 턴 기반 기업 경영 루프는 플레이 가능한 상태입니다.
- 경제 시스템(수요, 경쟁, 재고, 경기, 정책, 세금, 금융, 블랙스완)은 실코드에 반영되어 있습니다.
- 메뉴, 저장/이어하기/복구, 플레이 이력, 메타 보너스, 무한 모드가 연결되어 있습니다.
- 최근 작업으로 Phase 중심 UI, compact 레이아웃, hover 설명, 단일 결재 브리핑, 첫 플레이 튜토리얼이 추가되었습니다.
- 다음 단계의 핵심 과제는 “게임이 이해되느냐”보다 “중반 이후에도 계속 강해지고 싶어지느냐”입니다.

## 현재 설계 원칙

### 1. 교육 목표는 노출하지 않고 플레이로 체화시킨다
- 경제 개념을 직접 설명하는 문구보다, 가격·재고·부채·점유율 의사결정의 결과로 체감시키는 쪽을 우선합니다.
- 교과 성취기준은 개발 문서와 설계 근거에 남기되, 게임 내 표현은 경영/생존/확장 언어로 유지합니다.

### 2. 정보는 “전부 보여주기”보다 “지금 필요한 것 먼저”가 우선이다
- 한 화면에 많은 숫자가 있어도, 현재 우선 행동이 즉시 읽히지 않으면 UX가 무너집니다.
- 상단 요약, 접기/펼치기, compact 카드, hover 보조 설명을 유지·확장합니다.

### 3. 로그라이크답게 성장 곡선이 분명해야 한다
- 중반 이후가 지루해지지 않으려면 단순 숫자 증가가 아니라 “내 빌드가 무엇인지”가 드러나야 합니다.
- 다음 우선순위는 성장 HUD, 영구 특성, 중간 보상 이벤트, 빌드 시너지 강화입니다.

## 다음 우선순위

### P0. 성장감 강화
- [ ] 성장 HUD 추가
  - 산업 티어, 공장 레벨, 브랜드 등급, 신용등급, 현재 전략 축을 한눈에 보여주는 상시 UI
  - “다음 해금까지 남은 조건”을 함께 표기
- [ ] 런 내 영구 특성(빌드) 시스템
  - 예: 원가형 / 브랜드형 / 대량생산형 / 재무형
  - 특정 턴, 티어 해금, 공장 완공, 점유율 돌파 시 3지선다 보상 제공
- [ ] 빌드 시너지 강화
  - 보급형 + 공장 업그레이드 = 저원가 대량생산 루트
  - 프리미엄형 + 브랜드 누적 = 고가 전략 루트
  - 금융 안정 + 점유율 상승 = 공격적 확장 루트

### P1. 중반 이후 변주 확장
- [ ] 고정 시나리오 완화
  - 결재안, 정책 이벤트, 라이벌 압박 패턴의 조합 다양화
  - 산업 티어별 이벤트 풀 차등화
- [ ] 지역/채널/상품군 단위의 미세 분기 검토
  - 현재 아이템 탐색은 라인 수준이므로, 중반 이후는 세부 시장 차별화가 필요할 수 있음
- [ ] 보스 구간 설계
  - 37턴, 80턴, 111턴을 단순 텍스트 경고가 아니라 플레이 체감이 확실한 페이즈 전환으로 강화

### P2. 결재/이벤트 시스템 고도화
- [ ] 단일 결재안 1장 체계를 유지하되 카드 품질을 높이기
  - 카드 1장이면 선택 밀도는 올라가므로, 상황 반영성과 빌드 연계도를 더 강하게 만들어야 함
- [ ] 결재안 미리보기와 결과 리포트 연결 강화
  - “왜 이 카드가 떴는지”와 “무엇이 달라졌는지”를 요약하는 보조 문구 추가
- [ ] 후반 전용 결재안 풀 확장
  - Tier 3/4, 공장 레벨, 고브랜드/고부채 상태에 따라 카드군 차별화

### P3. 튜토리얼/온보딩 2차 개선
- [ ] 첫 플레이 튜토리얼을 상황 적응형으로 확장
  - 현재는 정적 UI 가이드 중심
  - 이후에는 “공급처 미선택”, “발주 미입력” 등 실제 상태 기반 안내를 추가
- [ ] 튜토리얼 종료 후 체크리스트 카드 제공
  - 첫 3턴 동안 “이번 턴 목표”를 간단히 다시 보여주는 보조 장치
- [ ] 접근성 보완
  - 키보드 포커스 이동, 버튼 라벨링, 튜토리얼 닫기/이전/다음 조작성 개선

### P4. 리포트/인텔 개선
- [ ] 우측 패널의 “현재 가장 위험한 라이벌” 표현 강화
- [ ] 리포트에서 성장 축(브랜드, 공장, 신용, 점유율) 변화를 더 눈에 띄게 표시
- [ ] 로그 압축/요약 뷰 검토
  - 정보량은 많지만 우선순위가 흐려질 수 있으므로 “핵심 3줄 요약” 도입 검토
