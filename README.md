# CapiRogue

경제 시뮬레이션, 전략 경영, 로그라이크 감각을 결합한 턴 기반 기업 운영 게임입니다.

플레이어는 제한된 자본으로 상품 라인을 고르고, 공급처를 계약하고, 가격과 발주를 조정하고, 매달 결재안과 거시경제 충격을 견디며 회사를 성장시켜야 합니다.

---

## 프로젝트 개요

CapiRogue는 “경제 개념을 설명하는 게임”보다는 “경영 판단을 반복하다 보면 경제 원리가 체화되는 게임”을 목표로 개발 중입니다.

핵심 방향은 다음과 같습니다.
- 경제 개념을 노골적으로 가르치기보다 플레이 결과로 체감시키기
- 매 턴 의사결정의 압박을 유지하되 UI는 점점 더 읽기 쉽게 정리하기
- 중반 이후에도 로그라이크처럼 성장 축과 빌드 감각이 살아 있도록 확장하기

---

## 현재 플레이 루프

현재 빌드의 기본 흐름은 아래와 같습니다.

1. 난이도와 시작 조건을 고른다.
2. 이번 달에 팔 상품/라인과 OEM 공급처를 정한다.
3. 품질 방향, 발주 수량, 판매가를 세팅한다.
4. 인사, 마케팅, 공장, 부동산, 금융, M&A 같은 추가 행동을 선택한다.
5. 실행 직전 결재안 1장을 확인하고 선택한다.
6. 월간 정산을 보고 다음 턴 전략을 조정한다.

최근 UI 개편으로 이 흐름은 `Phase 01 ~ 05` 구간으로 분리되어 있고, 완료된 구간은 접고 요약만 보면서 플레이할 수 있습니다.

## 구현된 핵심 시스템

### 경제/시장
- 가격 탄력 기반 수요 함수
- 난이도별 수요 민감도
- 거시경제 순환: `boom / stable / recession`
- 80턴 이후 블랙스완 이벤트
- 정책 이벤트(규제/보조금)
- 누진세, 신용등급, 실효금리, 연속 적자 패널티

### 경쟁
- 라이벌 4유형 아키타입
  - 저가 공세형
  - 프리미엄형
  - 변동성형
  - 고티어 기술 독점형
- 시장 점유율 경쟁 및 파산 후 재진입

### 생산/운영
- OEM 탐색 및 공급처 선택
- 공장 건설과 업그레이드
- 공장 완공 후 품질 모드 해금
- 공장 가동 후 라인 고정, 업그레이드 시 재선정 허용
- 재고 노화, 악성 재고, 강제 폐기

### UX/메타
- 스플래시, 메인 메뉴, 새로하기, 이어하기
- 저장 시점 복구와 플레이 이력
- 메타 보너스 누적
- 난이도 화면의 커스텀 세팅 및 화면 배율 조정
- 첫 플레이 튜토리얼 오버레이
- 설정 모달에서 튜토리얼 다시 보기

## 최근 UX 개선 포인트

### 정보 구조
- 지금 해야 할 행동을 상단 상황 요약 스트립으로 먼저 보여줍니다.
- Phase 01/02는 완료 시 자동으로 접히는 방향으로 구성되어 정보 과부하를 줄였습니다.

### 의사결정 보조
- 티어, 품질, 발주, 가격 프리셋에 hover 설명을 넣어 초보 플레이어가 개념 장벽 없이 선택할 수 있게 했습니다.
- 클릭 가능한 카드와 잠긴 카드의 hover 반응을 분리해 “지금 눌러도 되는가”를 바로 읽을 수 있게 했습니다.

### 결재 흐름
- 결재 브리핑은 이제 항상 1장만 미리 보여주고, 실제 실행 시에도 1장만 선택합니다.
- 실행 버튼 직전 문구와 브리핑 구조를 단순화해 흐름을 더 직관적으로 만들었습니다.

### 온보딩
- 첫 게임 진입 시 `localStorage["firstPlay"]` 기준으로 핵심 UI 영역 튜토리얼이 자동 실행됩니다.
- 튜토리얼은 설정 모달에서 다시 열 수 있습니다.

## 환경 변수

DeepSeek 연동을 사용할 경우 아래 값 중 하나를 설정합니다.

권장:
- `DEEPSEEK_API_KEY`

호환 fallback:
- `VITE_DEEPSEEK_API_KEY`

오프라인 폴백 공급처 생성 로직이 있어, 키가 없더라도 기본 플레이 테스트는 가능합니다.

---

## 저장 및 localStorage

현재 빌드는 여러 UI/진행 상태를 `localStorage`에 저장합니다.

- 런 저장/이어하기
- 복구 시점 목록
- 플레이 이력
- 메타 보너스
- UI 설정(BGM, 화면 배율)
- 첫 플레이 튜토리얼 여부 (`firstPlay`)

테스트 중 튜토리얼을 다시 보고 싶다면 설정 모달의 “튜토리얼 다시 보기”를 쓰거나, 브라우저에서 `firstPlay` 키를 지우면 됩니다.

## 현재 우선 과제

다음 단계의 가장 중요한 목표는 “중반 이후에도 지루하지 않은 성장감”입니다.

현재 계획은 아래 방향에 집중합니다.
- 성장 HUD 추가
- 런 내 영구 특성/빌드 시스템
- 중간 보상 이벤트와 해금 구조 강화
- 후반부 시나리오 변주 확대

자세한 우선순위는 `plan.md`를 기준으로 관리합니다.
